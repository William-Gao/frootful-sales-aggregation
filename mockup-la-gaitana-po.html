Design Notes

  • Conditional rendering — The richer table (Code/Item/Cases/Pack/$/UPC) only appears when po_metadata is present. All existing proposals continue using the simple Item/Size/Qty format.
  • PO metadata card — Sits between the customer/date bar and the items table. Shows PO identifiers (PO#, Cust PO, SO, Sell-to) and logistics (dates, ship method, consolidation location). HEB badge uses the existing green pill style.
  • Totals row — Only shown when po_metadata has totals. Cases + stems on the left, dollar amount right-aligned.
  • UPC column — 10px monospace font so the barcodes are readable but don't dominate the layout.
  • Editing — Cases column is editable (same number input as Qty today). Code, Pack, $/Unit, UPC are read-only display. Item name is still the searchable dropdown.
  • No breaking changes — All new fields are optional. Existing proposals work exactly as before.
